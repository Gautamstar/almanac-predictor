03 / Deep Dive

Long Short-Term Memory Networks

The Almanac uses a PyTorch LSTM model trained on S&P 500 historical data with walk-forward validation and MinMax normalization, achieving over 70% directional accuracy on held-out test windows.

Why LSTM for Stock Prices?

Stock prices are sequential, non-stationary time series. A standard feed-forward network treats each day's price independently, discarding temporal order entirely. Recurrent Neural Networks (RNNs) introduced memory, but suffer from the vanishing gradient problem: gradients shrink exponentially as they travel backward through time, making it impossible to learn dependencies spanning more than roughly 10 steps.

LSTMs solve this with a gated memory cell that selectively retains information across hundreds of timesteps. For S&P 500 prediction this matters because macro trends, earnings cycles, and sector rotations play out over weeks and months. The model was validated using walk-forward validation, where each training window advances one step at a time to simulate real deployment conditions and prevent data leakage. All price and volume features were scaled using MinMax normalization to the [0, 1] range before being fed into the network, keeping gradients stable across features with very different magnitudes.

Inside the LSTM Cell

Every timestep passes through three gates before updating the hidden state:

Forget Gate
ft = σ(Wf·[ht-1, xt] + bf)

Decides what portion of the previous cell state Ct-1 to discard. A value near 0 means "forget everything"; near 1 means "keep everything."

In market terms: when a company announces a major restructuring, the forget gate zeroes out learned patterns from a completely different operating regime.

Input Gate
it = σ(Wi·[ht-1, xt] + bi)
C̃t = tanh(Wc·[ht-1, xt] + bc)

Controls what new information gets written into the cell state. The candidate vector C̃t proposes new values; the gate decides how much of that proposal survives.

A breakout above a 52-week high produces a large candidate value that the input gate amplifies into the cell state.

Cell State Update
Ct = ft ⊙ Ct-1 + it ⊙ C̃t

The cell state is the LSTM's long-term memory, acting as a conveyor belt that carries information across many timesteps with minimal distortion and avoiding the vanishing gradient issue of plain RNNs.

Output Gate
ot = σ(Wo·[ht-1, xt] + bo)
ht = ot ⊙ tanh(Ct)

Filters the cell state to produce the hidden state ht, the short-term output passed both to the next cell and to the prediction head.

Only the features relevant to the current prediction step (e.g., recent momentum) are surfaced, while long-term baselines stay in Ct.

Input Features

Each timestep feeds a 7-column feature vector drawn from the Azure SQL gold layer. All numeric features are scaled to [0, 1] using MinMax normalization before being passed to the network:

Column Type Role in the Model
Symbol Categorical S&P 500 ticker identifier. Label-encoded before training.
Date Temporal Trading date. Sorted ascending so each sequence respects chronological order.
Close_Prices Float (normalized) Primary prediction target and input feature. MinMax-scaled to [0, 1].
High_Prices Float (normalized) Intraday high; contributes to volatility signals in the hidden state.
Low_Prices Float (normalized) Intraday low; combined with High to compute the normalized trading range.
Open_Prices Float (normalized) Opening price; the gap between Open and prior Close is a key momentum signal.
Volume Float (normalized) Share volume normalized independently. High volume confirms directional moves.

The backend queries the last 30 trading days from the gold layer, sorts ascending by date, applies the fitted MinMax scaler, and writes a temporary CSV before passing the path to the model's predict() method.

The pre-trained model is deserialized from lstm_model.pkl using joblib.load(). LSTMModel is injected into sys.modules['__main__'] so joblib can locate the class during unpickling.

Output Dispatch

The prediction return type is introspected and dispatched:

  • None: linear regression fallback (30-day horizon)
  • scalar / 0-d array: linear interpolation from last close to predicted value
  • array-like: reversed and trimmed to 30 points, overlaid on chart
|
5
Chart JSON

Historical data and prediction arrays are merged into the response JSON. The prediction series is annotated with borderDash: [5, 5] (dashed red line) so users can distinguish forecasts from actuals.

Robust Fallback: Linear Regression

Production ML systems should fail gracefully. If the LSTM model returns None, raises an exception, or outputs an incompatible type, the backend automatically switches to scikit-learn's LinearRegression. It fits a trend line on the 30-day closing price history, then projects it 30 days forward, ensuring the user always receives a prediction and never a blank chart.

The fallback series is labeled "Fallback Predictions" and uses the same dashed red styling, maintaining visual consistency.

04 / Data Pipeline

Multi-Layer Data Pipeline

Raw market data travels through a three-tier Bronze-Silver-Gold medallion architecture built on Azure Data Factory and Databricks, reducing pipeline latency by 35% compared to the prior single-layer approach.

Bronze

Raw Ingestion

yfinance and Financial Modeling Prep API data is pulled by Azure Data Factory pipelines. Raw JSON and CSV payloads land in Azure Blob Storage without modification, preserving a complete audit trail of every upstream value.

yfinance / FMP API / Azure Data Factory / Azure Blob
to
Silver

Transform and Clean

Databricks notebooks validate schemas, cast types, deduplicate records, and apply business-logic transformations. SQL databases with live replication and automated backup store the cleaned datasets, with Azure Grafana providing health monitoring and performance tracking.

Databricks / Azure SQL / Azure Grafana
to
Gold

Serving and Visualization

Django views query the gold layer via pyodbc and route data into the PyTorch LSTM model for inference. Model results and business KPIs are surfaced through QuickSight and Power BI dashboards alongside the real-time web and mobile clients.

Machine Learning

  • MinMax Normalization: feature scaling to [0, 1] before training
  • scikit-learn: linear regression fallback predictor
  • pandas and NumPy: feature engineering and array operations

Backend

  • Python: primary language for model and pipeline code
  • Django: web framework and REST API layer

Data Pipeline

  • Azure Data Factory: ETL orchestration across all three tiers
  • Databricks: silver-layer transformation and cleaning notebooks
  • Azure Blob Storage: bronze raw data lake

  • Azure Grafana: automated health monitoring and performance tracking

Frontend

  • Django and Bootstrap: responsive web client

BI and Visualization
