01 / Overview

What is The Almanac Predictor?

The Almanac Predictor is a full-stack financial intelligence platform that combines deep learning, cloud databases, and interactive visualization to help users understand and anticipate stock price movements.

Stock Forecasting

LSTM-based deep learning model trained on historical OHLCV data, generating 30-day forward price projections with a linear regression fallback for robustness.

Interactive Dashboard

Four real-time chart types (line, price-change, bar, scatter) built with Chart.js. Select any ticker and see history plus predictions rendered instantly.

News Aggregation

Relevant financial news pulled from live APIs and surfaced alongside price data, giving context behind price movements.

Portfolio Management

Users can track a personal watchlist of stocks, review P/E ratios, book values, and OHLC metrics across their selected holdings.

Multi-Device Support

Responsive web client built with Bootstrap 5, plus a Flutter mobile app. One backend serves both platforms via a unified REST API.

Cloud-Native Infrastructure

Azure SQL Database as the silver-layer store, Azure Blob Storage for raw data, and Azure Data Factory orchestrating ingestion pipelines.

03 / Deep Dive

Long Short-Term Memory Networks

LSTM networks are the backbone of the Almanac Predictor's forecasting engine. Understanding why LSTMs excel at time-series data and how they are wired up here is key to understanding the whole system.

Why LSTM for Stock Prices?

Stock prices are sequential, non-stationary time series. A standard feed-forward network treats each day's price independently, discarding the temporal order entirely. Recurrent Neural Networks (RNNs) introduced memory, but they suffer from the vanishing gradient problem: gradients shrink exponentially as they travel backward through time, making it impossible to learn dependencies spanning more than roughly 10 steps.

LSTMs solve this with a gated memory cell that can selectively remember information across hundreds of timesteps. This is critical for stock data because a trend established 60 trading days ago can still be relevant to tomorrow's price.

Inside the LSTM Cell

Every timestep passes through three gates before updating the hidden state:

Forget Gate
ft = σ(Wf·[ht-1, xt] + bf)

Decides what portion of the previous cell state Ct-1 to discard. A value near 0 means "forget everything"; near 1 means "keep everything."

In market terms: when a company announces a major restructuring, the forget gate zeroes out learned patterns from a completely different operating regime.

Input Gate
it = σ(Wi·[ht-1, xt] + bi)
C̃t = tanh(Wc·[ht-1, xt] + bc)

Controls what new information gets written into the cell state. The candidate vector C̃t proposes new values; the gate decides how much of that proposal survives.

A breakout above a 52-week high produces a large candidate value that the input gate amplifies into the cell state.

Cell State Update
Ct = ft ⊙ Ct-1 + it ⊙ C̃t

The cell state is the LSTM's long-term memory, acting as a conveyor belt that carries information across many timesteps with minimal distortion and avoiding the vanishing gradient issue of plain RNNs.

Output Gate
ot = σ(Wo·[ht-1, xt] + bo)
ht = ot ⊙ tanh(Ct)

Filters the cell state to produce the hidden state ht, the short-term output passed both to the next cell and to the prediction head.

Only the features relevant to the current prediction step (e.g., recent momentum) are surfaced, while long-term baselines stay in Ct.

Input Features

Each timestep feeds a 7-column feature vector drawn from the Azure SQL silver layer:

Column Type Role in the Model
Symbol Categorical Stock ticker identifier (e.g., AAPL). Encoded before training.
Date Temporal Trading date. Sorted ascending so sequences respect chronological order.
Close_Prices Float Primary prediction target. Used as the label and also as an input feature.
High_Prices Float Intraday high; contributes to volatility signals in the hidden state.
Low_Prices Float Intraday low; combined with High to compute trading range.
Open_Prices Float Opening price; gap between Open and previous Close is a key momentum indicator.
Volume Float Number of shares traded. High volume often confirms price moves and the model uses this to weight trends.

The backend queries the last 30 trading days of data, sorts rows by date ascending, and writes a temporary CSV to disk before passing the path to the model's predict() method.

Prediction Sequence

1
Query

Django fetches TOP 30 rows for AAPL from StockPriceSilverData_Table, ordered newest-first, then re-sorted ascending.

|
2
Serialize to CSV

The DataFrame is written to a temp file at %TEMP%/stock_data_for_model.csv. The model interface expects a file path rather than an in-memory array, keeping the training and inference interfaces identical.

|
3
Load and Infer

The pre-trained model is deserialized from lstm_model.pkl using joblib.load(). LSTMModel is injected into sys.modules['__main__'] so joblib can locate the class during unpickling.

|
4
Output Dispatch

The prediction return type is introspected and dispatched:

  • None: linear regression fallback (30-day horizon)
  • scalar / 0-d array: linear interpolation from last close to predicted value
  • array-like: reversed and trimmed to 30 points, overlaid on chart
|
5
Chart JSON

Historical data and prediction arrays are merged into the response JSON. The prediction series is annotated with borderDash: [5, 5] (dashed red line) so users can distinguish forecasts from actuals.

Robust Fallback: Linear Regression

Production ML systems should fail gracefully. If the LSTM model returns None, raises an exception, or outputs an incompatible type, the backend automatically switches to scikit-learn's LinearRegression. It fits a trend line on the 30-day closing price history, then projects it 30 days forward, ensuring the user always receives a prediction and never a blank chart.

The fallback series is labeled "Fallback Predictions" and uses the same dashed red styling, maintaining visual consistency.

04 / Data Pipeline

Multi-Layer Data Pipeline

Raw market data travels through a medallion-style architecture (bronze, silver) before reaching the prediction engine. Each layer adds validation, normalization, and business-logic transformations.

Bronze

Raw Ingestion

yfinance and Financial Modeling Prep API data is pulled by Prefect-orchestrated Python tasks. Raw JSON/CSV payloads land in Azure Blob Storage without modification, giving a complete audit trail of every upstream value.

yfinance / FMP API / Prefect / Azure Blob
to
Silver

Cleaned and Typed

Azure Data Factory pipelines validate schemas, cast columns to correct SQL types, deduplicate records, and load into Azure SQL Database. The primary tables are StockPriceSilverData_Table and Financial_Statement_Historical_Dimensional_Table.

Azure Data Factory / Azure SQL / Polars
to
Serving

API and Model Input

Django views query the silver layer via pyodbc, construct pandas DataFrames, and route data into either the LSTM model or the chart-data endpoints. Results are serialized as JSON and consumed by the web and mobile clients.
